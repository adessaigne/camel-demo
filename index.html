Concepts

Message based

It uses Exchange objects

An input, an output and meta-data (headers and properties).

URI based

bean://myBean?method=transform

  • bean defines the component
  • myBean defines the endpoint
  • method=transform defines the options

Routing system

The message goes from a source through several transformations using a routing system.

Example: filter, aggregator, enricher, normalizer...

Chaining

  • From a Consumer created by an Endpoint
  • To a Producer created by an Endpoint
  • To other Producers

Expressions

Extract information using several languages

~ 20 languages

Data formats

Encodes and decodes data using formats

~ 40 formats

Type conversion

Powerful built-in system for type conversion

Easily extensible

Registry

Built-in registry for finding objects

Can be backed by JNDI or OSGi

Extensible

Create custom components, languages, data formats...

The code

Using java DSL

DefaultCamelContext context = new DefaultCamelContext();
context.addRoutes(new RouteBuilder() {
    @Override
    public void configure() throws Exception {
        from("file:inbox?move=.processed")
                .unmarshal().csv()
                .split().body()
                .setHeader("CamelJmsMessageType").constant(JmsMessageType.Map)
                .to("jms:queue:order?connectionFactory=activemq");
    }
});
context.start();

Using Spring DSL

<camelContext id="camel" xmlns="http://camel.apache.org/schema/spring">
    <route>
        <from uri="file:inbox?move=.processed" />
        <unmarshal>
            <csv/>
        </unmarshal>
        <split>
            <simple>${body}</simple>
            <setHeader headerName="CamelJmsMessageType">
                <groovy>org.apache.camel.component.jms.JmsMessageType.Map</groovy>
            </setHeader>
            <to uri="jms:queue:order?connectionFactory=activemq"/>
        </split>
    </route>
</camelContext>
