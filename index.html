Apache Camel

The basics

by Antoine DESSAIGNE

What is Apache Camel ?

It's an open source library that

  • Consumes data
  • Tranforms data
  • Produces data

Really good for data integration

What can it consumes ?

Lot's of things. Really, no kidding.

Files, JMX events, JMS messages, SQL database rows, NoSQL databases values, SOAP web services, SOAP requests, REST web services, REST requests, FTP servers, mails, sockets, HTTP pages, Atom feeds, RSS feeds, web sockets, Docker, ElasticSearch, Spring events, OSGI events, system commands, Facebook, Google drive, IRC, Jira, LinkedIn, PaxLogging, Quartz events, RMI, Salesforce, SAP NetWeaver, LDAP, SSH, I/O Streams, Twitter, XMPP, ...

What can it consumes ?

Nearly 150 consumers are officially supported

You can also write your own consumers :)

What about data transformation ?

It's whatever you want.

Using data formats

CSV, fixed-width, TSV, XML, JSon, Zip, PGP, base64, protobuf, badly-formatted XML, ...

Official list available here.

You can also write your own particular formats

Applying operations

Content based routing, filtering, dynamic routing, recipient list, split, aggregation, resequencer, normalizer, content filter, sort

Official list available here.
