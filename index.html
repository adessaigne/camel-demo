Apache Camel

The basics


by Antoine DESSAIGNE

What is Apache Camel ?

It's an open source library

  • It consumes data from various sources
  • It routes data through many transformations
  • It produces data to any destination

Really good for data integration

Concepts


A bit of theory

Consumers

Read data

Depend on the source itself

Referenced by an URI

file://inbox?move=.processed

What are the consumers available ?

A lot. Really, no kidding!

Files JMS messages SQL data
NoSQL data SOAP services SOAP requests
REST services REST requests FTP
Mails Sockets JMX events
HTTP pages HTTP requests Atom/RSS feeds
Salesforce Facebook Twitter
and many more... and yours ☺

Nearly 150 consumers are officially supported

Producers

Chained after a consumer

Chained together

Creates data based on their input

Referenced by an URI

string-template://mail

What are the producers available ?

Many more than consumers. True.

Local files Remote files JMS messages
SQL data NoSQL data SOAP responses
REST responses Mails HTTP responses
and many more... and yours ☺

More than 150 producers are officially supported

What's inside a Camel Exchange ?

  • an input message
    • with body (any Object)
    • with headers (a Map<String, ?>)
  • an output message with body and headers
  • properties (a Map<String, ?>)
